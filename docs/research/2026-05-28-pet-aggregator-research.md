## 1. Мировые аналоги — лучшие практики

**Petfinder (US, powered by Purina):**
- Крупнейшая в мире БД adoptable pets, обслуживает 14 000+ приютов БЕСПЛАТНО. Доход — спонсорство Purina + 10% комиссия с платформы Sponsor-a-Pet.
- Premium для приютов: расширенные фильтры (порода, life-stage), аналитика просмотров/запросов, приоритетное размещение.
- Жалобы пользователей в 2024-2025: путаная навигация, неактивные приюты, мало фотографий, перегруженность фильтрами. Типовой failure mode зрелого классифайда.

**Adopt-a-Pet.com (501c3 nonprofit, EIN 45-0404057):**
- 15 000+ приютов, 300 000+ животных, 5M+ посетителей/месяц. Полностью БЕСПЛАТНО для приютов.

**Pets4Homes (UK, коммерческий маркетплейс):**
- 26 000+ животных. Hybrid: и приюты, и breeders, и owners. Поиск, мессенджер внутри платформы, alerts по saved-search.

**RescueMe.org:** простейшая open-database модель — любой может постить бесплатно. Минимум барьеров → максимум UGC → проблемы с модерацией.

**Tierheim.de / Balu (getbalu.com):** Германия — 350+ приютов агрегируются в одном UI.

**PetRescue.com.au:** 1 000 000 успешных пристройств. Чёткая mission-driven позиционировка работает лучше нейтрального "маркетплейса".

**Общие паттерны UX/UI:**
- Фильтры: вид, размер, возраст, пол, локация (главное!), сроки пребывания в приюте, статус прививок/стерилизации.
- Saved searches + email alerts.
- Высокое качество фото — сильнейший конверсионный фактор.
- "Адопшн-флоу" как разговор: пара вопросов → matching → знакомство.

**Бизнес-модель:** Petfinder = корпоративный спонсор + 10% с донатов. Adopt-a-Pet = гранты + донаты. Pets4Homes = платные размещения от breeders, freemium для приютов.

## 4. Платёжные системы РБ для донатов

**ЕРИП ("Расчёт")** — обязательный канал:
- 13 000+ точек оплаты по стране, 24/7. Подключают через агрегаторов (не напрямую как физлицо).
- Базовая комиссия: 0,2% на благотворительные платежи.
- **Обязательно благотворительный счёт в банке** — нужно юр.лицо.

**Агрегаторы:**
| Сервис | Комиссия НКО | Особенности |
|---|---|---|
| **bePaid** | 1% + ЕРИП по тарифу | Минимум 0.5 БВ/мес |
| **WebPay** | До 2.9% | Подключение 1 БВ |
| **ExpressPay** | от 0,3% (зависит от оборота) | QR, Telegram/Viber-уведомления |
| **hutkigrosh** | свободный формат для НКО | Нерегулярные платежи |
| **Assist Belarus** | — | Классический эквайринг |

**SMS/USSD донаты:**
- *222*7# — единый USSD.
- МТС/life:): SMS на 505.
- A1: SMS на 553 (2 BYN).

**Stripe/PayPal:** недоступны для приёма платежей юр.лицами РБ.

**Crypto:** USDT TRC20 возможно через ПВТ. Для широкой аудитории — не нужно.

## 5. Базы чипов животных

**Беларусь:**
- **animal-id.by** / **animalid.by** — единая база чипированных РБ. Код страны 112. С 2015.
- Синхронизация с **petmaxx.com** ~7 дней после имплантации.
- Проверка по 15-значному коду через веб-форму.
- **Публичного API нет** — нужен договор с support.

**Международные lookup:**
- **AAHA Universal Pet Microchip Lookup** — https://www.aaha.org/your-pet/pet-microchip-lookup/faqs/
- **PetMaxx** — https://www.petmaxx.com/
- **EuroPetNet** — европейская сеть.
- **Findefix** (Германия) и **TASSO.net** — крупнейшие в ЕС, бесплатные.

**Стандарт:** ISO 11784/11785, 15-значный код.

**Что делать в проекте:**
1. MVP — простая форма "введите 15 цифр" → редирект на animal-id.by с предзаполненным значением (или iframe).
2. Fallback для не-белорусских чипов — PetMaxx.
3. Долгосрочно — запросить partner API у animal-id.by.

**Важно:** в РБ чипирование **не обязательно для всех** домашних. Только для границ ЕС, охотничьих, выставок. Для бездомных чипа обычно нет. Фича lookup полезна, но не центральная.

## 6. Простота для пользователей с низкой цифровой грамотностью

**Nielsen Norman Group — low-literacy:**
- Lower-literacy users **plowing**, не сканируют. Пропускают элементы из-за узкого поля внимания.
- Упрощённая навигация. Сравнения без удержания в short-term memory.
- Plain language, короткие предложения, активный залог.
- Большие изображения вместо иконок.

**GOV.UK Design System:**
- WCAG 2.2 AA обязательный с октября 2024.
- "Один вопрос — одна страница" в формах. Прогресс-бар.
- Plain language как часть accessibility.

**Конкретные паттерны:**
- Большие кнопки (мин 44x44px) с понятными глаголами: "Помочь", "Взять домой", "Сообщить".
- Минимум обязательных полей.
- Один экран — один вопрос для критических флоу.
- Голосовой ввод через native HTML dictation.
- Помощь по шагам — встроенный walkthrough.
- Иконки + текст, не "только иконки".
- Контрастность 4.5:1 (WCAG AA).

**Telegram Mini App vs PWA vs веб:**
| Критерий | TG Mini App | PWA | Полный веб |
|---|---|---|---|
| Барьер входа | минимальный (уже в TG) | низкий | низкий |
| Зависимость от платформы | TG обязателен | нет | нет |
| Offline | ограниченно | да | нет |
| SEO/поисковый трафик | нет | да | да |
| Сложность для соло | средняя | средняя | низкая |

**Рекомендация:** **основа — адаптивный веб-сайт + Telegram-бот** как дополнительный канал. SEO критичен для агрегатора.

## 7. Юр.блок — НПА РБ и генерация документов

**Действующий закон:**
- **Закон РБ № 361-З "Об ответственном обращении с животными"** от 1 апреля 2024, в силе с 1 января 2025.
- Закон 144-З "О животном мире" (10 июля 2007 № 257-З) — дикая природа.
- Ноябрь 2025 — публичное обсуждение поправок.

**Уголовная — ст. 339-1 УК РБ:**
- Жестокое обращение → общественные работы, штраф, исп.работы до 1 года, арест.
- Повторно/группой → арест/лишение свободы до 3 лет.
- Возраст — с 16 лет.

**Административная — ст. 16.29 КоАП РБ:**
- Жестокое обращение без гибели → штраф до 15 БВ.
- Возраст — с 14 лет.

**Куда подавать:**
- МВД (РОВД) или местный орган управления.
- Единый портал https://обращения.бел онлайн.
- Лично/почтой/через дежурную часть. Телефон 102.
- Помощь — https://azo.by/.

**Существующие сервисы:**
- **ilex.by** — платный, образцы документов, ЭЦП.
- **etalonline.by** — НЦПИ, "Конструктор документов".
- **azo.by** — заявления о жестоком обращении.
- **helpsafe.by** — блок-схема "куда обращаться".
- **vetpravo.by** — ветеринарная экспертиза.

**Для MVP юр.раздела:**
1. Текстовые карточки "Куда обращаться".
2. Mail-merge шаблоны: пользователь заполняет 5-6 полей → готовый PDF/DOCX.
3. Подборка ссылок на действующие НПА (pravo.by/etalonline.by).
4. FAQ по ст. 339-1 УК и 16.29 КоАП.
5. **НЕ оказывайте платную юр.помощь** без лицензии.

## 8. Стек-варианты реализации

### Вариант A — No-code/Low-code (Tilda + Airtable + Make + ExpressPay)

**Плюсы:** Запуск за 2-4 недели без бэкенд-навыков. Дёшево на старте.

**Минусы:** Webflow CMS — потолок 20 000 записей. Airtable rate-limit. 100k посетителей/мес — no-code начинает сыпаться. Невозможно реализовать chip-lookup, генератор юр.документов, модерацию UGC, чаты.

**Вердикт:** Подходит ТОЛЬКО для validation landing page. Не production.

### Вариант B — Next.js + Postgres + Prisma + Railway + S3

**Плюсы:**
- Один фуллстек справляется.
- Отличный SEO (SSR/SSG) — критично для агрегатора.
- TypeScript end-to-end через Prisma.
- Railway уже используется в текущем проекте — есть опыт деплоя.
- AI-помощники лучше работают с TS/React.

**Минусы:**
- Готового CMS под животных нет — headless (Payload, Sanity, Strapi) или своя админка.
- Стоимость хостинга растёт быстрее, чем у Django.
- Меньше built-in (нет встроенного admin/auth).

**Вердикт:** **Рекомендованный выбор**, особенно с опытом Next.js/Railway. Payload CMS даёт self-hosted admin "из коробки".

### Вариант C — Django + Postgres + Wagtail CMS + Cloudflare R2

**Плюсы:**
- Django Admin "из коробки" — модерация UGC без своего UI.
- Wagtail — отличный CMS для контентного сайта (статьи, юр.раздел, FAQ).
- Mature ecosystem: django-allauth, django-storages, drf, modeltranslation.
- Дёшево хостить (один VPS тянет 100k посетителей/мес).
- Один Python-разработчик справляется лучше, чем с двумя языками.

**Минусы:**
- SEO требует усилий (server-rendered HTML + правильная разметка).
- Templates Django — менее современно.
- Без HTMX/Alpine — интерактивность слабовата.

**Вердикт:** **Сильный кандидат №2**, если разработчик любит Python.

**Рекомендация:** **Next.js 14+ App Router + Postgres + Prisma + Payload CMS + Railway + Cloudflare R2 + Cloudflare DNS/CDN**.

2. Прямые кнопки на Telegram/Viber приюта — конкретные животные.
3. Whereby embed для редких видеоконсультаций.
4. Никакого форума.

## 10. Парсинг данных существующих организаций

**Легальность РБ:**
- **Закон РБ № 455-З "Об информации"**, ст. 16: общедоступная информация — сбор не противоречит закону.
- **Закон РБ № 262-З "Об авторском праве"** — тексты, фото защищены.
- **Закон РБ № 99-З "О защите персональных данных"** + Указ № 422 от 28.10.2021. Имена/контакты владельцев — персональные данные, нужно согласие.

**Этические правила:**
- Соблюдать robots.txt.
- Rate-limit.
- ToS платформ. Instagram блокирует, VK мягче.
- Не публиковать персональные данные без согласия.

**Здоровая альтернатива — партнёрство:**
1. Простая форма "Подключить приют": контакт + ссылка VK/TG/IG.
2. Двусторонний интерес: трафик в обе стороны.
3. **3 модели интеграции:**
   - Manual posting (приют сам через форму или Telegram-бот — у вас уже есть похожий).
   - Semi-auto: приют шлёт Excel/CSV/Google Sheets, импорт.
   - Full integration: webhook/API подписка на VK-группу (с согласия).
4. Юридически — договор-оферта: "Размещая объявление, вы передаёте неисключительные права...".

**Чего НЕ делать:**
- НЕ парсить чужие сайты без согласия.
- НЕ парсить персональные данные владельцев из VK.

## Критические инсайты

1. **Изучить ByPet до запуска** — bypet.by уже позиционируется как "первая благотворительная площадка". Если они работают — партнёрство или дифференциация (агрегатор vs донат-площадка). Слепо строить параллельно — самая дорогая ошибка.

2. **Партнёрство победит парсинг.** ~20 организаций + сотни волонтёров. Объехать за 2-3 месяца, подписать договорённости. Это эксклюзивный контент, доверие, защита от юр.рисков.

3. **ЕРИП + SMS-донаты — обязательный минимум, но обе требуют юр.лица.** План А: партнёрство с Эгидой/ЗООшанс как "техплатформа". План Б: своя НКО через 2-3 мес после MVP.

4. **Чип-lookup — фича, а не центр.** В РБ обязательность ограничена, у бездомышей чипов нет. Простая интеграция с animal-id.by (форма → редирект), а не 2 месяца на PetMaxx API.

5. **Phasing:**
   - **Фаза 1 (3 мес):** агрегатор животных с фильтром по городу, профили приютов, простые донаты (банк.реквизиты + ЕРИП через ExpressPay), юр.раздел = статьи + ссылки. Стек: Next.js + Postgres + Payload CMS + Railway. Tawk.to.
   - **Фаза 2 (4-6 мес):** генератор шаблонных юр.заявлений, чип-lookup, интеграция Telegram-бота (Pet BOT уже есть).
   - **Фаза 3 (6+ мес):** видеоконсультации, расширенная аналитика, чат пользователь↔приют.
   - **НЕ делайте:** форум, нативное мобильное, crypto, AI-matching — преждевременная оптимизация.
