# Pet Help Web — спецификация

Параллельный с Telegram-ботом Pet BOT веб-проект для волонтёров и людей, которые хотят помочь животным найти дом. Общей БД с ботом нет — единственная точка интеграции на первом этапе — Telegram-уведомления о заявках через отдельного бота-нотификатора.

## Ключевые решения

| Параметр | Решение |
|---|---|
| Связь с Pet BOT | Параллельный продукт, общей БД нет |
| География | РФ + РБ, нужен фильтр по стране/городу (на этапе каталога) |
| Язык | Только русский |
| Масштаб | До 200 карточек с запасом роста |
| Хостинг | Netlify |
| Команда | Соло-разработчик + AI |
| Админка | Только по приглашениям (этап 3) |
| Донаты | Реквизиты без провайдера, номер карты с кнопкой копирования |
| Уведомления о заявках | Telegram через отдельного бота-нотификатора |
| Визуальный стиль | Утверждается в Figma, в коде — нейтральный с CSS-переменными |
| Контент на старте | Заглушки в JSON/TS-файлах |

## Аудитория и сценарии

### Потенциальный усыновитель

1. Заходит на главную → видит свежие карточки
2. Открывает карточку питомца
3. Жмёт «Хочу забрать» → заполняет короткую форму
4. Получает подтверждение, волонтёр связывается с ним

### Волонтёр / приют

1. Получает приглашение-ссылку от администратора (этап 3)
2. Входит в админку
3. Создаёт карточку питомца: фото, анкета, статус
4. Модерирует входящие заявки на усыновление
5. Отмечает питомца как «пристроен» и публикует в «Истории успеха»

### Донатор

1. Открывает раздел «Помочь»
2. Видит номер карты и список актуальных нужд
3. Копирует номер карты одним нажатием
4. Переводит средства напрямую, без провайдера

### Помощник

1. Открывает раздел «Стать помощником»
2. Выбирает форму: передержка / транспорт / выгул / ветпомощь
3. Оставляет контакты и описание возможностей
4. Заявка приходит волонтёрам в Telegram

## Этап 1 — Лендинг

### Задачи этапа

Сделать один деплоябельный одностраничник на Netlify, который:

- рассказывает о проекте и мотивирует помочь
- показывает карточки-заглушки питомцев для превью каталога
- принимает заявки на усыновление и помощь через Telegram-уведомления
- позволяет в один клик скопировать номер карты для доната
- хорошо выглядит на мобилке (первичная аудитория)
- готов к замене визуального стиля после Figma-утверждения

### Стек

| Слой | Решение | Обоснование |
|---|---|---|
| Framework | Next.js 15 (App Router) + TypeScript | SSR, SEO, OG-превью, хорошо ложится на Netlify через `@netlify/plugin-nextjs` |
| Стили | Tailwind CSS 4 + CSS-переменные | Быстрая вёрстка, лёгкая замена темы после Figma |
| Иконки | lucide-react | Готовый набор, лёгкий |
| Шрифт | `next/font` (Inter + Manrope) | Локальная подгрузка, без FOIT |
| Формы | Netlify Forms + serverless-функция → Telegram Bot API | Бесплатно до 100 заявок/мес, спам-защита Netlify |
| Контент | Локальные JSON/TS-заглушки в `src/content/` | Редактируется без трогания кода |
| Валидация | Zod | Типобезопасные схемы форм |
| Деплой | Netlify (прод) + preview-деплои из PR | Дефолт для Netlify |

Почему отказались от альтернатив:

- Astro — проще, но Next даст больше гибкости на этапах каталога и админки
- FastAPI + Jinja — не ложится на Netlify естественно
- Headless CMS — избыточно для этапа заглушек

### Структура лендинга по секциям

#### Header (sticky на десктопе)

- Логотип-плейсхолдер (SVG-иконка лапы) + название «Pet Help»
- Якорная навигация: Питомцы · Истории · Помочь · Стать помощником · Контакты
- На мобилке — бургер-меню

#### 1. Hero

- Полноэкранный (на мобилке — 85vh)
- Заголовок: «Каждой лапе — свой дом»
- Подзаголовок: 1–2 предложения о проекте
- Две CTA-кнопки: «Посмотреть питомцев», «Помочь»
- Фоном — крупное фото животного

#### 2. About — о проекте

- Короткое описание (3–4 абзаца): кто мы, что делаем, чем отличаемся
- 3 карточки с цифрами-заглушками: пристроено, волонтёров, городов
- Упоминание Pet BOT как сопутствующего инструмента волонтёров

#### 3. Pets — превью питомцев

- Сетка 3 × 2 (desktop) / 1 колонка (mobile)
- Карточка: фото, кличка, вид, пол, возраст, короткое описание, статус
- Кнопка «Хочу забрать» открывает модалку Adopt с предзаполненным `pet_id`
- Фильтров на лендинге нет — это для этапа 2 (каталог)
- Внизу ссылка «Скоро — полный каталог»

#### 4. Stories — истории успеха

- Слайдер/grid из 3–4 историй
- Каждая: фото (или до/после), имя, история на 2–3 абзаца, дата пристройства

#### 5. Help — как помочь проекту

- Слева: номер карты (крупный, моноширинный) + кнопка «Скопировать» с тостом «Скопировано»
- Справа: список актуальных нужд с приоритетами
- Комментарий: «Все сборы — прямые переводы. Мы не используем платёжные провайдеры на этапе старта.»

#### 6. Volunteer — стать помощником

- Табы/селектор типа помощи: передержка / транспорт / вет / выгул
- Поля: имя, контакт, город, описание возможностей
- Netlify Forms + hidden `type`
- Отправка → `/api/submit` → Telegram

#### 7. Adopt — заявка на усыновление

- Модалка, открывается из карточки питомца
- Поля: имя, контакт, город, сообщение, hidden `pet_id`
- Отправка → Telegram

#### 8. Contacts

- Telegram-канал, Telegram-бот (Pet BOT для волонтёров), email, город
- Форма обратной связи (имя, сообщение)

#### Footer

- Короткие якорные ссылки
- Ссылка на политику конфиденциальности `/privacy`
- Год + тег-лайн

### Формы и интеграции

Поток отправки заявки:

1. Пользователь заполняет форму (Adopt / Volunteer / Contacts)
2. Netlify Forms принимает сабмит (встроенный honeypot + спам-фильтр)
3. Netlify вызывает serverless-функцию `/api/submit`
4. Функция валидирует вход через Zod, формирует сообщение и шлёт его в Telegram через `sendMessage` у бота-нотификатора
5. Если Telegram недоступен — запись остаётся в Netlify Forms (видно в UI Netlify)

Важно: бот-нотификатор — отдельный бот, созданный через BotFather. Он не связан с Pet BOT ни токеном, ни кодом, ни БД.

Формат сообщения в Telegram:

```text
🐾 Новая заявка — Усыновление

Питомец: Барсик (pet_id=barsik)
Имя: Иван
Контакт: @ivan_tg / +79991234567
Город: Минск
Сообщение: Есть опыт содержания кошек, готов забрать...

Источник: лендинг
```

### Дизайн-система

- CSS-переменные темы в `globals.css`:

  ```css
  :root {
    --color-primary: #e87a55;
    --color-primary-fg: #fff;
    --color-bg: #fdfaf6;
    --color-fg: #2a2118;
    --color-muted: #7a6f63;
    --radius: 1rem;
  }
  ```

- Все секции используют только эти переменные через Tailwind-токены — замена цвета после Figma сводится к правке 5 строк
- Шрифты: Manrope (заголовки), Inter (тело)
- Брейкпоинты: 640 / 768 / 1024 / 1280
- Radius: 1rem у карточек, 0.75rem у кнопок
- Анимации: `prefers-reduced-motion` respected, только fade/translate на 150–250мс

### Нефункциональные требования

- Lighthouse Mobile: Performance ≥ 90, Accessibility ≥ 95, Best Practices ≥ 95, SEO ≥ 95
- LCP ≤ 2.5 с на 4G
- Все фото через `next/image` с `loading="lazy"`, кроме hero (`priority`)
- Все формы имеют `aria-label`, видимые ошибки валидации, `Enter`-submit
- `alt` у каждого фото
- `<meta>` и OpenGraph теги на каждой странице
- `robots.txt` разрешает всё, `sitemap.xml` генерится автоматически

### Поэтапная реализация

Атомарные шаги, каждый — отдельный коммит:

1. Скелет проекта: `create-next-app` в `web/`, первый деплой на Netlify
2. Дизайн-токены: CSS-переменные, глобальные стили, шрифты, `Button`/`Input`
3. Layout: Header (с навигацией + бургер), Footer, базовый `layout.tsx` и meta
4. Hero + About: статические секции, фото-заглушка, цифры
5. Контент-заглушки: `src/content/*.ts` + типы
6. Секция Pets: сетка карточек из заглушек, модалка «Хочу забрать»
7. Секция Stories: слайдер/grid историй
8. Секция Help: карта + CopyButton + список нужд
9. Секция Volunteer: форма с переключателем типа
10. Секция Adopt: реализация модалки + валидация Zod
11. Секция Contacts: форма обратной связи + статические контакты
12. Serverless `/api/submit`: функция + отправка в Telegram
13. Интеграция форм с `/api/submit`, тосты об успехе/ошибке
14. Полировка: OG-изображение, favicon, `privacy.tsx`, проверка Lighthouse, SEO-мета
15. README.md + `.env.example`

### Критерии готовности этапа 1

- Лендинг деплоится на Netlify без ошибок
- Все секции из списка присутствуют и наполнены заглушками
- Формы отправляются → приходят в Telegram
- Кнопка «Скопировать номер карты» работает
- Lighthouse Mobile ≥ 90 / 95 / 95 / 95
- Нет ошибок в консоли и предупреждений Next.js
- README описывает переменные окружения и локальный запуск

### Что не входит в этап 1 (отложено)

- Каталог с фильтрами и страницами питомцев
- Админка по инвайту, роли, модерация

- CMS для волонтёров
- i18n, английская версия

- Личный кабинет усыновителя

## Пост-MVP этапы

### Этап 2 — Каталог питомцев

- Страница `/pets` с фильтрами (вид, пол, возраст, размер, город/страна)
- Страница отдельного питомца `/pets/[id]`
- Подключение Postgres (Neon) + Prisma ORM
- Хранилище фото: Cloudflare R2 или Selectel S3
- OG-превью для каждой карточки

### Этап 3 — Админка для волонтёров

- Вход по пригласительным ссылкам (без публичной регистрации)
- CRUD карточек, смена статуса, модерация заявок
- Роли: super-admin, moderator, volunteer
- Аудит действий

### Этап 4 — Расширения

- Раздел «Истории успеха» из БД
- Формы помощи в админке со статусами
- Подписка на появление нового питомца по критериям
- Онлайн-платежи (YooKassa / CloudPayments), если будет юрлицо
- Многоязычность (при выходе за РФ/РБ)

## Инженерные ограничения и правила

- Публичная регистрация отсутствует (только инвайты на этапе 3)
- Онлайн-платежи не реализуются в MVP
- Интеграции с Pet BOT нет (кроме отдельного бота-нотификатора)
- Секреты (`.env`), дампы БД и локальные фото не должны попадать в git
- Весь контент и интерфейс на русском языке
