# Pet BOT — Telegram-бот для волонтёров

## Назначение

Pet BOT помогает быстро собрать объявление о пристройстве животного из одного фото. Бот принимает фотографию кошки или собаки, проводит пользователя через короткую анкету, генерирует варианты новых интерьерных фото и несколько готовых текстов объявления, после чего позволяет выбрать одно или несколько фото, при необходимости вручную отредактировать выбранный текст и опубликовать результат в отдельный Telegram-канал.

## Что умеет текущая версия

- Принимать фото животного только как Telegram photo attachment
- Собирать анкету через FSM и inline-кнопки
- Генерировать 3 варианта изображения через OpenAI Images API
- Генерировать 5 текстов объявления через OpenAI Chat API
- Давать пользователю выбрать одно или несколько фото и один текст объявления
- Давать пользователю редактировать любой сгенерированный текст объявления прямо в Telegram-чате
- Публиковать выбранную комбинацию в отдельный Telegram-канал с номером питомца, кнопкой `помочь` и текстовыми атрибутами на каждом фото
- Восстанавливать последнюю пользовательскую сессию после рестарта процесса
- Перегенерировать материалы без повторного заполнения анкеты

## Стек

- **Python 3.11+**
- **aiogram 3.15.0** — Telegram Bot API, polling, FSM
- **openai 2.30.0** — `gpt-image-1` для изображений и `gpt-4o-mini` для текстов
- **Pillow 12.2.0** — нормализация и подготовка референсных изображений
- **python-dotenv 1.1.0** — загрузка конфигурации из `.env`

## Архитектура

### Модули

#### `bot.py`

Основной orchestration layer:

- создаёт `Bot` и `Dispatcher`
- описывает FSM-состояния анкеты
- принимает фото и ответы пользователя
- запускает генерацию изображений и текстов
- хранит последние сессии и текущие сгенерированные результаты
- позволяет редактировать сгенерированные описания без создания нового варианта
- добавляет текстовые атрибуты поверх публикуемых фото
- публикует выбранный результат в отдельный канал

#### `image_service.py`

Отвечает за генерацию изображений:

- нормализует исходное фото с учётом EXIF
- уменьшает исходник до безопасного размера
- строит два референса: полный кадр и focus crop
- собирает промпт по данным анкеты
- вызывает `client.images.edit(...)` с несколькими fallback-вариантами качества и fidelity

#### `text_service.py`

Отвечает за генерацию текстов:

- превращает анкету в структурированный профиль животного
- формирует промпт для `gpt-4o-mini`
- просит несколько текстов в разных стилях
- устойчиво разбирает ответ модели по `---`, нумерованным пунктам или абзацам

#### `config.py`

Централизованно загружает переменные окружения и валидирует обязательные секреты при старте приложения.

### Состояния FSM

Анкета построена как линейный сценарий со следующими состояниями:

1. `Form.photo`
2. `Form.name`
3. `Form.animal_type`
4. `Form.sex`
5. `Form.age`
6. `Form.size`
7. `Form.character`
8. `Form.health`
9. `Form.comments`
10. `Form.result`
11. `Form.edit_text`

Особенности:

- `/start` всегда сбрасывает текущее состояние и запускает сценарий заново
- имя и комментарий могут быть пропущены
- после ввода комментария или нажатия "Пропустить" бот переводит пользователя в `Form.result`
- кнопка `generate` обрабатывается уже из финального состояния
- `Form.edit_text` используется для ручного редактирования одного выбранного текста объявления

### Хранилища данных

#### FSM storage

- используется `MemoryStorage`
- хранит активную анкету текущего пользователя в памяти процесса

#### `latest_sessions`

Словарь `dict[int, dict]`, который хранит последнюю завершённую анкету пользователя, включая `photo_bytes`.

Назначение:

- пережить рестарт процесса
- позволить повторную генерацию без повторной загрузки фото

Технически:

- сериализуется в `latest_sessions.json`
- `photo_bytes` сохраняются в base64
- загружается при старте приложения

#### `latest_generated`

Эфемерное in-memory хранилище для текущих результатов генерации:

- `images`
- `texts`
- `selected_photo_indices`
- `selected_text_index`
- `last_post_key`

Это хранилище не пишется на диск. После рестарта процесса бот может восстановить анкету, но не уже сгенерированные кандидаты.

#### `pet_counter.txt`

Отдельный runtime-файл, в котором хранится следующий свободный номер питомца для публикации в канал.

Назначение:

- выдавать последовательные номера `№1`, `№2`, `№3` и далее
- не терять сквозную нумерацию после рестарта процесса

## Логика работы

### Основной пользовательский сценарий

1. Пользователь отправляет `/start`
2. Бот просит фото животного
3. Пользователь отправляет фото
4. Бот задаёт вопросы по анкете
5. Пользователь доходит до шага комментария
6. После ввода комментария или пропуска бот показывает кнопку `Генерировать`
7. При генерации бот сохраняет анкету в `latest_sessions`
8. Бот параллельно запускает:
   - `generate_images(photo_bytes, data, count=3)`
   - `generate_texts(data, count=5)`
9. Если обе части успешны, бот отправляет:
   - 3 фото-кандидата с кнопками `pick_photo:<index>`
   - 5 текстов-кандидатов, где под каждым текстом есть кнопки `pick_text:<index>` и `edit_text:<index>`
10. Пользователь может сначала выбрать текст или сначала фото
11. При необходимости пользователь нажимает `Редактировать`, отправляет новый текст и заменяет им существующий вариант по тому же индексу
12. Когда выбраны фото и описание, бот показывает кнопку `Отправить в канал`
13. После публикации бот отправляет пост в канал

### Генерация изображений

Текущая реализация ориентирована не на полную перерисовку животного, а на identity-preserving image edit.

Ключевые детали:

- модель: `gpt-image-1`
- на вход подаются один или два референса одного и того же питомца
- промпт жёстко требует сохранить морду, окрас, пропорции и индивидуальность животного
- фон, композиция и свет выбираются случайно из предопределённых наборов
- один прогон может сделать до `count` изображений, но сервис ограничен максимумом в 4 результата
- для каждого изображения используется fallback-цепочка:
  - multi-image + `input_fidelity=high` + `quality=medium`
  - single-image + `input_fidelity=high` + `quality=medium`
  - single-image + `input_fidelity=high` + `quality=low`
  - single-image + `input_fidelity=low` + `quality=medium`
- при сетевых ошибках делается повтор через более лёгкий вариант запроса

### Генерация текстов

Текущая реализация делает упор на готовый рекламный copy для волонтёров.

Ключевые детали:

- модель: `gpt-4o-mini`
- промпт включает:
  - кличку, если она есть
  - тип животного
  - пол
  - возраст
  - размер
  - характер
  - здоровье
  - дополнительные комментарии
- ожидается 5 разных текстов на русском языке
- если модель не вернула разделитель `---`, код пытается разобрать ответ по нумерации или пустым строкам

### Логика публикации в канал

Публикация происходит только когда у пользователя есть:

- выбран хотя бы один индекс фото
- выбранный индекс текста
- настроенный `TELEGRAM_SPECIAL_CHANNEL_ID`

Формат публикации:

1. Пользователь может добавить в подборку одно или несколько фото
2. Пользователь может выбрать любой текст или предварительно вручную отредактировать его
3. После выбора фото и описания бот показывает отдельную кнопку `Отправить в канал`
4. Перед отправкой бот назначает питомцу сквозной номер `№N`
5. На каждое публикуемое фото бот накладывает текст без плашки, внизу справа:
   - `Кличка`
   - `Возраст`
   - `Размер`
   - `Характер`
   - `Здоровье`
6. Цвет текста автоматически подбирается по яркости правой нижней части изображения; для читаемости используется тонкая контрастная обводка
7. В канал отправляются:
   - выбранное фото или медиагруппа из нескольких фото с текстовыми атрибутами поверх изображения
   - отдельное сообщение с заголовком вида `Имя №N` или просто `№N`, если клички нет
   - ниже заголовка выбранный текст объявления
   - inline-кнопка `помочь`
8. По кнопке `помочь` бот показывает popup с номером питомца, номером карты и телефоном

Защита от дублей:

- в `latest_generated["last_post_key"]` хранится последняя опубликованная комбинация `(photo_indices, text_index)`
- если пользователь повторно отправит тот же набор фото и то же описание, бот не опубликует дубль

### Редактирование текстов

После генерации каждый текст объявления отправляется одним сообщением с двумя inline-кнопками:

- `Выбрать` — помечает этот текст как активный для публикации
- `Редактировать` — переводит пользователя в `Form.edit_text`

Сценарий редактирования:

1. Пользователь нажимает `Редактировать` под конкретным текстом
2. Бот просит прислать исправленный вариант одним сообщением
3. Полученный текст сохраняется в тот же индекс внутри `latest_generated["texts"]`
4. Исходное сообщение с описанием обновляется на месте
5. Новый вариант сразу доступен для выбора и публикации, без создания отдельного дубликата

### Логика перегенерации

Кнопка `Перегенерировать` работает от последней сохранённой сессии:

- сначала бот пытается взять данные из `latest_sessions`
- если там ничего нет, использует данные текущего FSM

Два режима:

1. Если тексты уже есть в `latest_generated`, бот перегенерирует только изображения и сохраняет уже существующие тексты.
2. Если текстов нет, бот перегенерирует и изображения, и тексты одновременно.

Это нужно для сценариев:

- обычная повторная попытка получить более удачные фото
- восстановление после частичного сбоя, когда в прошлый раз одна из частей не сгенерировалась

### Обработка ошибок и частичных сбоев

Если генерация вернула только часть результата:

- бот не оставляет "полусломанный" набор в `latest_generated`
- пользователю показывается кнопка `Перегенерировать`
- состояние остаётся в `Form.result`
- анкета и фото сохраняются, поэтому повторный запуск можно сделать без повторного ввода

Это закрывает проблемный сценарий, когда были фото без текстов или тексты без фото и пользователь не мог завершить публикацию.

## Конфигурация

### Обязательные переменные `.env`

```env
TELEGRAM_BOT_TOKEN=<токен от @BotFather>
OPENAI_API_KEY=<ключ OpenAI API>
```

### Переменные для публикации

```env
TELEGRAM_SPECIAL_CHANNEL_ID=<id или @username нового канала>
CONTACT_PHONE=<контактный телефон>
CARD_NUMBER=<номер карты для донатов>
```

Если `TELEGRAM_SPECIAL_CHANNEL_ID` не задан, бот всё равно может генерировать результаты, но не сможет публиковать выбранный пост в новый канал.

Для стабильной работы рекомендуется использовать числовой `chat_id` канала вида `-100...`, а не `@username`.

## Запуск

```bash
pip install -r requirements.txt
python bot.py
```

## Инженерные ограничения и правила

- Бот публичный, без whitelist по Telegram ID
- Используется polling, webhook не реализован
- База данных отсутствует
- Активный FSM живёт в памяти процесса
- Последняя пользовательская анкета хранится в `latest_sessions.json`
- Сгенерированные кандидаты не переживают рестарт процесса
- Интерфейс бота полностью на русском языке
- Секреты и runtime-данные не должны попадать в git:
  - `.env`
  - `latest_sessions.json`
  - `pet_counter.txt`
